# Agent Skills domain routing

Use this pattern when the ingested content's lasting value is a reusable method for agent-assisted engineering.

## Route here when
- The source describes a repeatable workflow for planning, coding, review, or compounding knowledge.
- The note is more about *how agents should work* than about one vendor, one release, or one transient tool setup.

## Example pattern from session
A Compound Engineering article describing Every's four-step process belonged in `Agent Skills`, not a generic AI/company bucket, because the durable value was the methodology:
- brainstorm / clarify
- plan
- execute
- review / compound

## Required graph updates
After routing into `Agent Skills`, update:
- the topic page for the method/concept
- related implementation/plugin page if one exists
- `wiki/index.md`
- `wiki/log.md`
- stable link-target stubs for people/org names when backlinks need canonical targets

## Durable lesson
When methodology and vendor are mixed, classify by the enduring operator value. The workflow usually matters more than the publisher.
