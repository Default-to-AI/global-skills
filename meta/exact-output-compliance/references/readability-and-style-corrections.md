# Readability and style corrections

## Trigger from this session
The user rejected two bad fallback shapes in sequence:
1. long bullet-chain answers
2. dense wall-of-text answers

The accepted shape was:
- short titled sections
- selective emoji for quick skimming
- concise paragraphs
- visible structure with some voice/body
- very restrained bullet use

## Operational rule
When a user says variants of:
- "stop this format"
- "too verbose"
- "too many bullets"
- "this is a block of text"
- "boring / unreadable"
- "remember this style"

then the agent should not merely shorten the answer. It should change the **information architecture** of the reply.

## Good target shape
Use a structure like:
- headline / framing line
- 3–6 short titled sections
- 1–3 short paragraphs each
- emojis only as scan anchors, not decoration spam
- bold only for the core nouns or verdicts

## Anti-patterns
- summary-framework sludge
- mirrored bullet sections saying the same thing in different headings
- giant prose block with no landmarks
- emoji-free monotone when the user explicitly wants skimmability
